# 4) خیلی مهم: `.env` را نباید عمومی کنی

فایل `.env` شامل اطلاعات حساس است:
- توکن ربات
- آیدی ادمین
- شاید بعداً اطلاعات دیگر

## قانون مهم
- برای اجرای محلی: داشتن `.env` خوب است
- برای GitHub عمومی: **نباید** `.env` را آپلود کنی

## در این پروژه دو روش بهتر داری

### روش امن برای GitHub
از فایل زیر استفاده کن:
- `.env.example`

و مقدارها را در پنل هاست، داخل **Environment Variables** وارد کن.

### روش محلی
از فایل `.env` استفاده کن.

# 5) ساخت GitHub برای اولین بار

اگر GitHub نداری:

1. وارد سایت GitHub شو
2. روی Sign up بزن
3. ایمیل و رمز عبور بساز
4. ثبت‌نام را کامل کن

بعد از ورود:
1. بالا سمت راست روی عکس پروفایل بزن
2. `Your repositories`
3. `New`
4. یک اسم برای مخزن بگذار، مثلاً:
   - `trexu-bot`
5. Public یا Private را انتخاب کن
   - اگر مبتدی هستی، بهتر است **Private** باشد
6. روی `Create repository` بزن

# 6) آپلود پروژه روی GitHub

دو راه داری:

## راه A: از خود سایت GitHub
اگر با git بلد نیستی، این راه ساده‌تر است.

### قدم‌ها
1. پروژه را روی کامپیوتر یا گوشی unzip کن
2. وارد ریپو جدید GitHub شو
3. `Add file` → `Upload files`
4. **همه فایل‌ها را آپلود کن، به‌جز `.env`**
5. منتظر بمان تا آپلود کامل شود
6. پایین صفحه `Commit changes` را بزن

## راه B: با git
اگر می‌خواهی حرفه‌ای‌تر کار کنی:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin YOUR_REPO_URL
git push -u origin main
```

> چون `.gitignore` داخل پروژه گذاشته شده، فایل `.env` در حالت عادی وارد git نمی‌شود.

# 7) دیپلوی روی Koyeb — آموزش قدم‌به‌قدم

## قدم 1: ساخت حساب
1. برو به Koyeb
2. ثبت‌نام کن
3. ایمیل را تأیید کن

## قدم 2: وصل کردن GitHub
1. وارد Dashboard شو
2. گزینه Create App یا New App را بزن
3. GitHub را انتخاب کن
4. اجازه دسترسی بده
5. ریپویی که ساختی را انتخاب کن

## قدم 3: انتخاب نوع سرویس
چون این ربات Polling است، بهتر است سرویس را **Worker** تنظیم کنی.

اگر از تو پرسید چه چیزی اجرا شود، این را بزن:

```bash
python bot2.py
```

## قدم 4: Build Command
اگر Koyeb خودش از فایل‌ها تشخیص نداد، این Build Command را بگذار:

```bash
pip install -r requirements.txt
```

## قدم 5: Environment Variables
این قسمت خیلی مهم است.
باید متغیرهای `.env` را یکی‌یکی وارد کنی.

حداقل این‌ها را وارد کن:

- `BOT_TOKEN`
- `ADMIN_IDS`
- `DATABASE_URL`
- `CHANNEL_URL`
- `SUPPORT_USERNAME`
- `BANK_CARD_NUMBER`
- `BANK_CARD_HOLDER`
- `FREE_DAILY_LIMIT`
- `FREE_AD_INTERVAL`
- `DOWNLOAD_WORKERS`
- `DEFAULT_LANG`

اگر AudD و Genius داری، این‌ها را هم وارد کن:
- `AUDD_API_TOKEN`
- `GENIUS_API_TOKEN`

## قدم 6: DATABASE_URL
برای شروع ساده، حتی می‌توانی SQLite را بگذاری:

```env
DATABASE_URL=sqlite:///trexu.db
```

ولی برای سرویس ابری، بهتر است PostgreSQL باشد.

اگر Koyeb یا سرویس دیگر PostgreSQL داد، شبیه این می‌شود:

```env
DATABASE_URL=postgresql+psycopg2://USER:PASSWORD@HOST:5432/DBNAME
```

## قدم 7: دیپلوی
روی Deploy بزن.

اگر همه‌چیز درست باشد:
- Build شروع می‌شود
- بعد Service بالا می‌آید
- Bot شروع به Polling می‌کند

## قدم 8: چک کردن لاگ
در پنل Koyeb بخشی برای Logs هست.
اگر خطا دیدی، همان‌جا مشخص می‌شود.

# 12) اگر خواستی VPS واقعی لینوکسی بگیری

اگر بعداً VPS واقعی Ubuntu گرفتی، راه‌اندازی کلی این شکلی است:

## قدم 1: اتصال SSH
```bash
ssh root@YOUR_SERVER_IP
```

## قدم 2: نصب پکیج‌ها
```bash
apt update && apt upgrade -y
apt install -y python3 python3-venv python3-pip ffmpeg git tmux
```

## قدم 3: گرفتن پروژه
```bash
git clone YOUR_REPO_URL trexu_bot_project
cd trexu_bot_project
```

## قدم 4: ساخت venv
```bash
python3 -m venv .venv
source .venv/bin/activate
```

## قدم 5: نصب requirements
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## قدم 6: ساخت `.env`
```bash
cp .env.example .env
nano .env
```

## قدم 7: اجرا
```bash
python bot2.py
```

## قدم 8: اجرای دائمی با tmux
```bash
tmux new -s trexu
python bot2.py
```

بعد برای جدا شدن:
- `Ctrl + b`
- `d`

# 13) خطاهای رایج روی سرور رایگان

## خطا: `BOT_TOKEN is required`
یعنی در Environment Variables توکن را نزده‌ای یا غلط وارد کرده‌ای.

## خطا: `ffmpeg not found`
یعنی محیط هاست ffmpeg ندارد.
باید Docker یا روش نصب سیستمی را فعال کنی.

## خطا: بعضی لینک‌ها دانلود نمی‌شوند
ممکن است نیاز به این‌ها داشته باشی:
- `PROXY_URL`
- `YT_DLP_COOKIE_FILE`

## خطا: سرویس بالا می‌آید ولی ربات جواب نمی‌دهد
این‌ها را چک کن:
- توکن درست است؟
- ربات از BotFather بلاک یا stop نشده؟
- Logs چه می‌گویند؟
- آیا سرویس واقعاً Running است؟
